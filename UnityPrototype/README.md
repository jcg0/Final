## How to Import SVG Files in Unity

Use these steps to import and use the `Assets/Art/*.svg` files included in this prototype:

1. Open your Unity project.
2. Go to **Window → Package Manager**.
3. In Package Manager, switch dropdown to **Unity Registry**.
4. Search for **Vector Graphics**.
5. Click **Install** (package name is typically `com.unity.vectorgraphics`).
6. Drag/copy this repo's `UnityPrototype/Assets/Art` folder into your Unity project's `Assets/` folder (or use **Assets → Import New Asset...**).
7. Click an imported `.svg` file in Project view and check Inspector:
   - **Texture Type**: `Sprite (2D and UI)`
   - **Pixels Per Unit**: choose a consistent value for your project (for example 100)
   - **Generate Physics Shape**: optional
8. Click **Apply** in Inspector.
9. Use the SVGs in scene/prefabs by dragging them onto SpriteRenderer components (2D) or UI Image components (UI).

### Common Issues

- **SVG imports as Default Asset / not visible**
  - The Vector Graphics package is not installed yet, or Unity needs a reimport.
  - Fix: install package, then right-click SVG → **Reimport**.

- **Sprites look blurry or too large/small**
  - Adjust **Pixels Per Unit** in Inspector and click **Apply**.

- **Package not found in Unity Registry**
  - Ensure you are online and using a Unity version compatible with the Vector Graphics package.

## Recommended Implementation Order (files)

Implement the prototype in this order so dependencies stay simple:

1. `Assets/Scripts/HexTile.cs`
   - Base terrain enum + coordinate component used by almost every other script.
2. `Assets/Scripts/HexBoard.cs`
   - Board indexing + adjacency methods; depends on `HexTile`.
3. `Assets/Scripts/OverlordUnit.cs`
   - Overlord movement/summon constraints; depends on `HexTile`.
4. `Assets/Scripts/MonsterUnit.cs`
   - Monster movement; depends on `HexTile` and `HexBoard`.
5. `Assets/Scripts/GameControllerPart1.cs`
   - Core gameplay flow; depends on board + units + tile setup.
6. `Assets/Scripts/CameraEdgeScroller.cs`
   - Camera control (independent gameplay-wise, but easier to tune once map exists).
7. `tools/generate_placeholder_assets.py`
   - Optional tooling for regenerating placeholder SVG assets.

## Unity Step-by-Step (detailed)

### 1) Create project and import files
1. Open Unity Hub and create a **3D** project.
2. Copy `UnityPrototype/Assets` into your Unity project `Assets/` directory.
3. If SVGs do not import, install package:
   - **Window → Package Manager → Unity Registry → Vector Graphics**.

### 2) Build hex tile prefab and map
1. Create one tile object (e.g., `Tile_Base`) with:
   - Mesh/Sprite renderer (your visual)
   - Collider (required for click raycasts)
   - `HexTile` component
2. In `HexTile`, set:
   - `q` and `r` axial coordinates
   - `tileType` for each terrain/building
3. Duplicate to build the map.
4. Ensure castle starts are valid:
   - each player start tile is `Castle`
   - each start castle has at least **3 adjacent** `Fortress` and/or `Tower` tiles

### 3) Create board root
1. Create empty GameObject named `Board`.
2. Add `HexBoard` component to `Board`.
3. Parent all hex tile objects under `Board`.

### 4) Create unit prefabs
1. Create `Overlord_1` prefab:
   - visual object
   - `OverlordUnit` component
   - set `playerId` (e.g., 1)
2. Create `Overlord_2` prefab:
   - `OverlordUnit` component
   - set `playerId` (e.g., 2)
3. Create `Monster_1` prefab:
   - visual object
   - `MonsterUnit` component
   - keep `movementRange = 1` for part 1

### 5) Create game controller object
1. Create empty GameObject named `GameController`.
2. Add `GameControllerPart1` component.
3. Assign references in Inspector:
   - `mainCamera` (or leave null to use `Camera.main`)
   - `board` (`Board` object)
   - `overlordChoices` (size 2: `Overlord_1`, `Overlord_2`)
   - `monsterPrefab` (`Monster_1`)
   - `playerOneCastle` (tile object)
   - `playerTwoCastle` (tile object)

### 6) Configure camera (top-down)
1. Select Main Camera.
2. Add `CameraEdgeScroller`.
3. Confirm/default settings:
   - `enforceTopDown = true`
   - `fixedHeight` to desired map height
   - `xBounds` / `zBounds` to match map size
   - `panSpeed` and `edgeThickness` tuned to preference
4. Press Play and verify camera stays top-down (`X=90, Y=0, Z=0`).

### 7) Playtest part-1 rules
1. **Left Click** tile with overlord/monster to select.
2. **Right Click** destination to move selected unit.
3. **Space** to summon monster from selected overlord.
4. Verify rule behavior:
   - overlords only move on castle/fortress/tower
   - summon only works on castle/fortress/tower
   - one monster per player max
   - monster moves one adjacent hex only

## Controls

- **Camera move (top-down):** mouse at screen edges or arrow keys
- **Select unit:** Left click a tile containing overlord/monster
- **Move selected:** Right click destination tile
- **Summon monster:** `Space` (when an overlord is selected)
